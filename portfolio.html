Designing CI/CD Using Jenkins, GitHub, SonarQube and Docker

I ever design a CI/CD Pipeline using Jenkins that integrated with GitHub, SonarQube and Docker. The explanation is here :

  • The Developer push the code to the GitHub.
  • If any modification code form GitHub, automatically, Jenkins will pull code to the server.
  • Jenkins will build the pulled code.
  • Jenkins will be testing the code using SonarQube.
  • If the code contain a lot of bugs, Jenkins will send email notification to the Developer.
  • If the code is OK to use, Jenkins will deploy to the container Docker from the server and send email notification SUCCESS.
